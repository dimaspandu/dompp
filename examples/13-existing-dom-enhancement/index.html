Existing DOM Enhancement

DOM++ can enhance existing server-rendered or static HTML without recreating the tree. Unlike setChildren(), setEnhancement() does not replace child nodes. Instead, it adopts the existing DOM structure and attaches behavior directly onto it.

Important:

setEnhancement() does not rerender or replace the existing DOM tree.

When state changes, the enhancement callback reruns and directly executes native DOM setters again.

This means mutation granularity depends entirely on which setters are called.

Examples:
  • setText() only updates text nodes
  • setStyles() only mutates inline styles
  • setAttributes() only mutates attributes
  • setEvents() only reattaches listeners
  • setChildren() performs full structural replacement
DOM++ does not use virtual DOM diffing or hidden reconciliation systems.
