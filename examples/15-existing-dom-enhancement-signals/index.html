Existing DOM Enhancement + Signals

DOM++ can enhance existing server-rendered HTML while still using optional fine-grained signals. This combines: - existing DOM adoption - zero structural replacement - signal-based granular updates

Important:

Unlike setChildren(), setEnhancement() does not recreate or replace the DOM tree.

Existing HTML becomes the reactive UI.

Only setters consuming signals rerun.
  • setText() updates only text nodes
  • setStyles() updates only inline styles
  • No virtual DOM is used
  • No structural reconciliation occurs
